# Elixis Agency Website Operations

This repo contains the Elixis Agency landing page for the pest-control AI receptionist offer. The website is served from `public/` by the same Express app used for the Retell backend, but website files must never contain backend secrets.

## Live Site

- Production domain: `https://elixis.agency`
- Vercel project: `retell-pest-control-demo`
- GitHub repo: `https://github.com/rizefr/RetellDemo`
- Root Directory: `./`
- Production branch: `main`

## How Deployment Works

- Previous manual deploys can show source `vercel deploy`.
- The current deployment goal is GitHub-based: commit to `main`, let Vercel build from `rizefr/RetellDemo`, then verify `https://elixis.agency`.
- GitHub-connected deploys should show commit and branch information instead of only `vercel deploy`.
- If a GitHub deployment fails with `Could not read package.json` at `/vercel/path0/package.json`, the GitHub repo root is missing the actual project files. Because Root Directory is `./`, `package.json`, `vercel.json`, `public/`, `src/`, and `web/` must be committed at the repository root.
- `Ready` means the deployment built successfully.
- `Error` means the deployment failed.
- `Production` means that deployment is currently assigned to the production domain.
- Short codes such as `HkweHsC9u` are deployment IDs or aliases, not branch names.
- The custom domain `elixis.agency` points to the latest promoted production deployment.

## Root Directory Setting

Vercel Root Directory `./` means Vercel expects the app at the repo root. Because the root is `./`, files like `package.json`, `vercel.json`, `public/`, `src/`, and `web/` should live at the top level of the GitHub repo.

Do not put the whole project inside an extra nested folder unless the Vercel Root Directory setting is changed to match that nested folder.

## Important Website Files

- `public/index.html`: landing page markup and copy.
- `public/site-config.js`: public, non-secret website URLs and labels.
- `public/site.js`: mobile menu, section scrolling, Retell modal, config application, reveal animation, and Cal.com embed bootstrapping.
- `web/styles.css`: Tailwind v4 source CSS and custom design system.
- `public/styles.css`: compiled CSS generated by `npm run build:styles`.
- `public/polish.css`: small production-safe override layer for the focused landing-page polish.
- `public/images/`: Elixis logo assets.
- `public/server.ts`: Vercel Express entrypoint. Do not remove unless the backend is intentionally moved elsewhere.

## Transcript Carousel

The hero demo console uses a controlled transcript carousel in `public/site.js`.

- There are exactly 3 pest-control scenarios.
- Each scenario renders exactly 4 transcript bubbles.
- The bubbles reveal once per scenario, one after another, then the complete conversation remains visible for about 6 seconds before the carousel advances.
- The 3 dots under the transcript manually switch scenarios and replay that scenario's reveal.
- Visitors with reduced-motion enabled see all 4 messages immediately without reveal animation.

## Current Page Structure

The live page is intentionally ordered around one product demo and one primary conversion path:

1. Hero and AI receptionist demo console
2. Missed, overflow, and after-hours coverage
3. Why it matters, with three response-focused statistic cards
4. How it works in 7 days
5. Bonus pest-control visibility audit
6. Embedded booking calendar

The hero is the only full product-demo section and positions the offer as a full AI receptionist. Missed, overflow, and after-hours language belongs in the coverage section below the hero. The footer contains the Elixis Agency contact number and both public contact emails.

To update site copy or section order, edit `public/index.html`. To update layout and responsive styling, edit `web/styles.css`, then run `npm run build:styles` to regenerate `public/styles.css`.

## How To Change The AI Demo URL

The public Retell orb URL lives in:

```text
public/site-config.js
```

The key to update is:

```js
AI_DEMO_ORB_URL: "https://agent.retellai.com/orb/..."
```

Preferred command:

```bash
npm run set:demo-url -- "https://agent.retellai.com/orb/NEW_AGENT_ID?token=PUBLIC_TOKEN"
```

The script validates that the URL starts with `https://agent.retellai.com/orb/` and refuses Retell API or dashboard URLs. It updates only `public/site-config.js` and never touches `.env`.

Local-only testing override:

```text
http://localhost:3000/?demoOrbUrl=https%3A%2F%2Fagent.retellai.com%2Forb%2FNEW_AGENT_ID%3Ftoken%3DPUBLIC_TOKEN
```

This override only works on localhost-style hosts and only affects the current browser session. It does not persist to production files.

Never paste Retell API keys, Cal.com API keys, Supabase keys, or private white-label tokens into frontend files.

## Booking Links

Current public booking defaults live in `public/site-config.js`:

- `BOOKING_URL`: external fallback booking page.
- `CAL_PUBLIC_BOOKING_URL`: public Cal.com event link used by the inline embed.
- `CAL_EVENT_TYPE_ID`: reference ID for the Cal event type.

If a new Cal event type needs to be resolved, do that server-side or in Cal.com, then put only the public booking URL in `public/site-config.js`.

## Retell Branding Limitation

The visible Elixis page UI says `Talk to AI`. Text inside the Retell-hosted iframe is controlled by Retell and cannot be safely changed from parent-page CSS or JavaScript because the iframe is cross-origin. Use Retell-supported white-label/widget settings if internal iframe branding needs to change.
